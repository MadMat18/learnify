R203 - Services réseaux

Introduction au DHCP

Le protocole DHCP (Dynamic Host Configuration Protocol) est un protocole de réseau qui permet aux ordinateurs et aux autres appareils d'obtenir automatiquement une adresse IP et d'autres informations réseau telles que le masque de sous-réseau, la passerelle par défaut et les serveurs DNS.

Fonctionnement

Lorsqu'un ordinateur se connecte à un réseau, il envoie une requête DHCP pour demander une adresse IP. Le serveur DHCP du réseau reçoit cette requête et attribue une adresse IP disponible à l'ordinateur.

En plus de l'adresse IP, le serveur DHCP peut également fournir d'autres informations de configuration, telles que le masque de sous-réseau, la passerelle par défaut et les serveurs DNS.

Avantages

Le DHCP permet une gestion simplifiée des adresses IP sur un réseau, en évitant les conflits d'adresses IP et en permettant une allocation dynamique des adresses IP. Il permet également une configuration automatique des paramètres réseau des ordinateurs connectés, ce qui simplifie la tâche de l'administrateur réseau.

Inconvénients

Le DHCP peut poser un risque pour la sécurité du réseau si des personnes malveillantes parviennent à se faire passer pour des clients DHCP légitimes et obtiennent des adresses IP valides. De plus, le DHCP nécessite une configuration correcte pour fonctionner efficacement sur le réseau.

Processus de location d'une adresse IP via DHCP

  1. Découverte DHCP : le client envoie une requête de diffusion (broadcast) à l'ensemble du réseau pour trouver un serveur DHCP disponible.
  2. Offre DHCP : les serveurs DHCP répondent avec une offre qui contient une adresse IP disponible et d'autres informations de configuration.
  3. Requête DHCP : le client accepte l'offre DHCP et envoie une demande pour obtenir l'adresse IP proposée.
  4. Attribution DHCP : le serveur DHCP confirme l'attribution de l'adresse IP et envoie les autres informations de configuration telles que le masque de sous-réseau, la passerelle par défaut et les serveurs DNS.

Types de serveurs DHCP

Il existe deux types de serveurs DHCP : les serveurs DHCP "standalone" et les serveurs DHCP "relay". Les serveurs DHCP standalone sont directement connectés au réseau et peuvent répondre aux demandes DHCP directement. Les serveurs DHCP relay sont utilisés lorsque les clients DHCP se trouvent sur un réseau différent de celui où se trouve le serveur DHCP. Le serveur DHCP relay agit comme un intermédiaire en relayant les requêtes DHCP du client vers le serveur DHCP et en renvoyant la réponse du serveur DHCP vers le client.

Le protocole DHCP est largement utilisé dans les réseaux locaux (LAN) pour automatiser la configuration des adresses IP. Il offre un moyen facile et efficace de gérer les adresses IP dans un réseau de grande envergure.

Les serveurs web et le protocole HTTP en réseau

Un serveur web est un programme informatique qui permet de stocker et de distribuer des pages web. Ces pages sont généralement écrites en langage HTML, CSS et JavaScript. Lorsqu'un utilisateur demande une page web à un serveur web, celui-ci répond en envoyant la page demandée à l'utilisateur via le protocole HTTP (HyperText Transfer Protocol).

Le protocole HTTP est un protocole de communication client-serveur qui permet d'échanger des données sur le Web. Il permet de transmettre des fichiers multimédias tels que des images, des vidéos, des fichiers audio, etc. Le protocole HTTP est basé sur une architecture client-serveur. Le client est généralement un navigateur web et le serveur est le programme qui stocke et distribue les pages web.

Le fonctionnement d'un serveur web

Le fonctionnement d'un serveur web est basé sur une architecture client-serveur. Lorsqu'un utilisateur demande une page web, le navigateur envoie une requête au serveur web. Le serveur web reçoit cette requête, recherche la page demandée et renvoie la page au navigateur en utilisant le protocole HTTP. Le navigateur reçoit la page et l'affiche à l'utilisateur.

Le serveur web utilise généralement un système d'exploitation tel que Linux ou Windows. Il utilise également des logiciels tels que Apache, Nginx ou IIS pour stocker et distribuer les pages web. Ces logiciels sont des serveurs web populaires qui sont utilisés par de nombreuses entreprises et organisations dans le monde entier.

Le protocole HTTP

Le protocole HTTP est un protocole de communication client-serveur. Le client envoie une requête au serveur en utilisant le protocole HTTP et le serveur renvoie une réponse. Les requêtes et les réponses HTTP sont composées de messages.

Le message HTTP est constitué de deux parties : l'en-tête (header) et le corps (body). L'en-tête contient des informations sur la requête ou la réponse, telles que le type de contenu, la taille du fichier, etc. Le corps contient les données réelles envoyées dans la requête ou la réponse. Par exemple, le corps peut contenir une page web, une image, un fichier audio, etc.

Le protocole HTTP est un protocole sans état, ce qui signifie que le serveur web ne garde pas de mémoire des requêtes précédentes. Chaque requête est traitée de manière indépendante, sans aucune connaissance des requêtes précédentes. Pour les applications Web qui nécessitent des informations de session, les développeurs utilisent généralement des cookies ou des sessions pour stocker ces informations.

Le protocole DNS en réseau

Le DNS (Domain Name System) est un protocole réseau permettant de convertir un nom de domaine en adresse IP. Il permet ainsi de simplifier l'accès à des ressources réseau en utilisant des noms de domaines plus facilement mémorisables que des adresses IP.

Fonctionnement du DNS

Le DNS est organisé en une hiérarchie de serveurs. Les requêtes DNS sont envoyées à un serveur DNS local, qui consulte les serveurs DNS racines pour obtenir la réponse. Les serveurs DNS racines renvoient alors le nom et l'adresse IP du serveur DNS du domaine correspondant. Le serveur DNS local peut ensuite contacter directement le serveur DNS du domaine pour obtenir l'adresse IP correspondant au nom de domaine recherché.

Types d'enregistrements DNS

Il existe différents types d'enregistrements DNS pour spécifier la correspondance entre un nom de domaine et une adresse IP. Les plus courants sont :

  • A : permet de spécifier l'adresse IP d'un hôte
  • CNAME : permet de créer un alias pour un nom de domaine
  • MX : permet de spécifier le serveur de messagerie pour un domaine
  • NS : permet de spécifier le serveur DNS pour un domaine

Sécurité du DNS

Le DNS est un protocole sensible aux attaques car il permet à un attaquant de rediriger le trafic vers un serveur malveillant. Pour se protéger contre ces attaques, il est recommandé d'utiliser DNSSEC (DNS Security Extensions), qui permet de garantir l'authenticité et l'intégrité des réponses DNS.
